# Post - Comment API 설계 정리

## 기본 관계

- Post : Comment = 1 : N
- 외래키(FK)는 Comment가 가짐
  - `comment.post_id`

Comment는 Post에 종속된 하위 리소스이다.

---

# 1. Create (댓글 작성)

### URL
```bash
POST /posts/{post_id}/comments
```
### Body
```json
{
  "content": "댓글 내용"
}
```
### 설명
- 댓글은 특정 게시글에 속함
- post_id는 URL에서 받음
- comment_id는 서버에서 자동 생성

# 2. Read (조회)
댓글은 단독 조회하지 않고, 게시글 상세 조회 시 함께 반환.

### URL
```bash
GET /posts/{post_id}
```
```json
{
  "post_id": 1,
  "title": "게시글 제목",
  "comments": [
    {
      "comment_id": 1,
      "content": "댓글1"
    },
    {
      "comment_id": 2,
      "content": "댓글2"
    }
  ]
}
```
# 3. Update (댓글 수정)
### 권장 방식
```bash
PATCH /comments/{comment_id}
```
### Body
```json
{
  "content": "수정된 댓글"
}
```
### 설명
- comment_id는 전역적으로 유니크
- 부모 리소스(post_id)를 굳이 포함하지 않아도 됨

계층형 URL (선택)
```bash
PATCH /posts/{post_id}/comments/{comment_id}
```
# 4. Delete (댓글 삭제)
### 권장 방식
``` bash
DELETE /comments/{comment_id}
```
